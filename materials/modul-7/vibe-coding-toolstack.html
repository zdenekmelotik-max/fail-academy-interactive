01

Tři nástroje, tři vrstvy

Kód potřebuje místo, kde žije, místo, kde běží, a místo, kam ukládá data. Každý nástroj řeší jednu vrstvu.

📦
GitHub
Verzování & spolupráce
Úložiště vašeho kódu s kompletní historií. Jako Google Docs pro kód — víte kdo, kdy a co změnil. Můžete se kdykoliv vrátit zpět.
🚀
Vercel / Railway
Deploy & hosting
Z kódu na GitHubu udělá živý web nebo aplikaci. Propojíte jednou, pak se každá změna automaticky publikuje. Žádné servery, žádná konfigurace.
🗄️
Supabase / Neon
Databáze & backend
Kam si aplikace ukládá data — uživatele, obsah, nastavení. Bez databáze máte jen statickou stránku. S ní máte plnohodnotnou aplikaci.

Klíčové: Všechny tři nástroje mají štědrý free tier. Na prototyp a MVP nepotřebujete žádný rozpočet — jen účet a 30 minut na setup.

03

GitHub & Git

Verzování kódu je jako „uložit hru" — můžete se vrátit k libovolnému bodu. GitHub je místo, kde váš kód žije online a kde na něm můžete spolupracovat.

Co to řeší lidsky
  • Máte kompletní historii — kdo, kdy a co změnil
  • Pokud něco rozbijete, vrátíte se o krok zpět
  • Víc lidí (nebo AI agentů) může pracovat současně
  • Kód je zálohovaný v cloudu, ne jen na vašem počítači
  • Sdílíte odkaz na repo místo posílání souborů
Technické minimum
  • Git = nástroj na vašem počítači, verzuje soubory
  • GitHub = cloud, kde repo žije online
  • Commit = „uložení" s popisem co se změnilo
  • Push = odeslání commitů na GitHub
  • Branch = experimentální kopie pro nové funkce
  • Pull Request = návrh na sloučení změn

Proč je to důležité pro vibe coding

Cursor, Claude Code a další AI nástroje pracují přímo s gitem. Když AI agent udělá změnu, automaticky ji commitne. Vidíte přesně co AI udělal, a pokud výsledek nefunguje, jednoduše se vrátíte. Bez gitu je AI agent jako spolupracovník, který maže za sebou poznámky.

Free tier

GitHub je zdarma pro neomezený počet public i private repozitářů. Pro jednotlivce a malé týmy nepotřebujete placený plán.

04

Vercel & Railway

Hosting nové generace — propojíte s GitHubem a každá změna v kódu se automaticky publikuje jako živý web. Žádné servery, žádné FTP, žádná konfigurace.

Co to řeší lidsky
  • Váš projekt je dostupný na internetu za sekundy
  • Každá změna na GitHubu = automatický nový deploy
  • Každý pull request dostane vlastní preview URL
  • Nepotřebujete rozumět serverům ani infrastruktuře
  • Škáluje se samo — od prototypu po produkci
Technické minimum
  • Deploy = nasazení kódu na server, aby byl web live
  • CI/CD = automatický build + deploy po každém push
  • Preview = dočasná URL pro testování změn
  • Environment variables = tajné klíče (API, DB)
  • Rollback = návrat k předchozí verzi jedním klikem

Vercel vs Railway — kdy co

Vercel

Optimalizovaný pro frontend a Next.js. Excelentní pro weby, landing pages, SaaS produkty. Nejrychlejší developer experience na trhu — import repo, klik, běží.

vercel.com · Free tier: 100 GB bandwidth

Railway

Univerzálnější — hostuje cokoliv: Python backend, Node API, databáze, cron joby. Ideální když potřebujete víc než jen frontend. Jednoduchý jako Vercel, ale flexibilnější.

railway.app · Free tier: $5 kredit / měsíc

05

Supabase & Neon

Databáze je úložiště vaší aplikace. Bez ní je web jen brožura. S ní je to produkt — s uživateli, obsahem a logikou.

Co to řeší lidsky
  • Ukládá uživatele, obsah, nastavení, objednávky...
  • Aplikace si „pamatuje" — data přežijí refresh
  • Přihlašování uživatelů (auth) je součástí
  • Data jsou strukturovaná a prohledávatelná
  • AI nástroje umí DB schéma navrhnout za vás
Technické minimum
  • PostgreSQL = typ databáze (průmyslový standard)
  • Tabulka = jako Excel sheet s definovanými sloupci
  • Connection string = adresa DB pro připojení z kódu
  • Auth = přihlašování, registrace, session
  • Row Level Security = kdo vidí která data

Supabase vs Neon — kdy co

Supabase

Postgres + Auth + Storage + Realtime v jednom balíku. „Firebase pro Postgres." Ideální když chcete kompletní backend bez psaní API — přihlašování, ukládání souborů, real-time notifikace.

supabase.com · Free: 2 projekty, 500 MB DB

Neon

Čistý serverless Postgres s branching (jako git pro databázi). Perfektní když potřebujete jen databázi a auth řešíte jinak. Scales to zero — neplatíte, když aplikace neběží.

neon.tech · Free: 0.5 GB storage, branching

06

Setup pro začátečníka

Kompletní stack za 30 minut. Všechno je zdarma, všechno nastavíte z prohlížeče. Pak už jen otevřete Cursor a začnete stavět.

  1. Založte si GitHub účet Jděte na github.com → Sign up. Zvolte free plán. Tohle je váš „cloud disk" pro kód.
  2. Nainstalujte Git na počítač Mac: už máte (otevřete Terminál, napište git --version). Windows: stáhněte z git-scm.com. Cursor i Claude Code git používají na pozadí.
  3. Vytvořte první repo na GitHubu GitHub → New repository → pojmenujte (např. my-first-app) → Create. Tohle bude domov vašeho projektu.
  4. Propojte Vercel s GitHubem Jděte na vercel.com → Sign up with GitHub → Import repository. Vercel se napojí a každý push na GitHub automaticky publikuje váš web.
  5. Založte databázi na Supabase Jděte na supabase.com → Start your project → propojte s GitHubem. Zkopírujte connection string do Vercel Environment Variables.
  6. Otevřete Cursor a začněte stavět Naklonujte repo z GitHubu do Cursoru. Řekněte AI co chcete postavit. Každý git push = nová verze na webu. Hotovo.

Výsledek: Máte kompletní profesionální stack — verzování, automatický deploy a databázi. Stejnou infrastrukturu používají startupy i korporáty. A nestálo to ani korunu.

07

Slovníček pojmů

Všechno, co uslyšíte při vibe codingu, na jednom místě. Bez zbytečného techno-žargonu.

Git

Nástroj na vašem počítači, který sleduje změny v souborech. Jako „uložit hru" — můžete se vrátit k libovolnému bodu.

Repository (repo)

Složka s projektem, kterou Git sleduje. Obsahuje kód i celou historii změn. Žije lokálně i na GitHubu.

Commit

Jedno „uložení" s popisem, co se změnilo. Jako záložka v knize — víte přesně, kde jste byli.

Push

Odeslání vašich commitů z počítače na GitHub. Jako „synchronizovat" v Dropboxu — lokální změny se nahrají do cloudu.

Pull

Stažení změn z GitHubu do počítače. Opak pushe — dostanete, co někdo jiný (nebo AI) změnil.

Branch

Paralelní kopie kódu pro experiment. Jako „uložit jako kopii" — zkoušíte nový nápad, aniž byste rozbili originál.

Pull Request (PR)

Žádost o sloučení vaší branch do hlavního kódu. Jako „schvalte prosím mou změnu" — ostatní můžou dát feedback.

Deploy

Nasazení kódu na server, aby byl web dostupný na internetu. Jako „publikovat" v blogu — z konceptu se stane živá stránka.

Lokální server

Web běžící jen na vašem počítači (localhost). Vidíte ho jen vy — ideální na testování, než změny publikujete.

Build

Překlad zdrojového kódu do podoby, kterou prohlížeč umí zobrazit. Jako „export do PDF" — z pracovního souboru se stane finální verze.

Environment Variables

Tajné hodnoty (API klíče, hesla k DB), které kód potřebuje, ale nesmí být viditelné v repozitáři. Jako trezor na hesla.

CI/CD

Automatická pipeline: push → build → deploy. Nemusíte nic ručně spouštět — kód jde z vašeho počítače rovnou na web.

Frontend

Vše, co uživatel vidí a na co kliká — webové stránky, tlačítka, formuláře. Běží v prohlížeči návštěvníka.

Backend

Logika běžící na serveru — zpracovává data, komunikuje s databází, řeší přihlašování. Uživatel ho přímo nevidí.

API

Rozhraní, přes které spolu mluví programy. Jako číšník v restauraci — přenáší objednávky (požadavky) mezi vámi a kuchyní (serverem).

Connection String

Adresa vaší databáze — obsahuje server, port, jméno a heslo. Jako celá URL adresa s přihlášením pro vaši DB.

Rollback

Návrat k předchozí fungující verzi webu. Když něco pokazíte, jedním klikem se vrátíte. Záchranná síť pro deploy.

npm / Package Manager

Nástroj na stahování hotových knihoven. Jako App Store pro kód — místo psaní od nuly nainstalujete ověřené komponenty.
